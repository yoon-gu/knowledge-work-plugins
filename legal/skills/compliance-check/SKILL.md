# /compliance-check -- 컴플라이언스 검토

> 익숙하지 않은 플레이스홀더가 있거나 연결된 도구를 확인하려면 [CONNECTORS.md](../../CONNECTORS.md)를 참조하세요.

제안된 조치, 제품 기능, 마케팅 캠페인, 또는 사업 이니셔티브에 대해 컴플라이언스 검토를 실행합니다.

**중요**: 이 명령은 법무 워크플로우를 지원하지만 법률 자문을 제공하지 않습니다. 컴플라이언스 평가는 자격을 갖춘 법률 전문가의 검토를 받아야 합니다. 규제 요건은 자주 변경되므로 항상 공식 소스에서 현행 요건을 확인하세요.

## 사용법

```
/compliance-check $ARGUMENTS
```

## 필요한 정보

계획하고 있는 내용을 설명해 주세요. 예시:
- "현금 보상이 있는 추천 프로그램을 출시하고 싶습니다"
- "모바일 앱에 생체 인증을 추가하려고 합니다"
- "미국 데이터 센터에서 EU 고객 데이터를 처리해야 합니다"
- "마케팅에서 고객 추천서를 광고에 사용하고 싶어합니다"

## 개인정보보호 규제 개요

### GDPR (일반 데이터 보호 규정)

**적용 범위**: 처리 조직의 소재지와 관계없이 EU/EEA 내 개인의 개인정보 처리에 적용됩니다.

**사내 법무팀의 주요 의무**:
- **적법한 근거**: 각 처리 활동에 대한 적법한 근거를 식별하고 문서화(동의, 계약, 정당한 이익, 법적 의무, 중대한 이익, 공적 과제)
- **정보주체 권리**: 접근, 정정, 삭제, 이동, 처리 제한, 이의 제기 요청에 30일 이내 응답(복잡한 요청의 경우 60일 연장 가능)
- **데이터 보호 영향 평가(DPIA)**: 개인에게 높은 위험을 초래할 가능성이 있는 처리에 필요
- **침해 통지**: 개인정보 침해를 인지한 후 72시간 이내 감독 당국 통지; 높은 위험이 있는 경우 영향 받는 개인에게 지체 없이 통지
- **처리 기록**: GDPR 제30조 처리 활동 기록 유지
- **국제 이전**: EEA 외부 이전에 적절한 보호 장치 확보(SCCs, 적정성 결정, BCRs)
- **DPO 요건**: 필요한 경우 데이터 보호 책임자 지정(공공 기관, 특수 카테고리의 대규모 처리, 대규모 체계적 모니터링)

**일반적인 사내 법무 접점**:
- 공급업체 DPA의 GDPR 준수 검토
- 제품팀에 프라이버시 바이 디자인 요건 자문
- 감독 당국 문의 대응
- 국경 간 데이터 이전 메커니즘 관리
- 동의 메커니즘 및 개인정보 처리방침 검토

### CCPA / CPRA (캘리포니아 소비자 프라이버시법 / 캘리포니아 프라이버시 권리법)

**적용 범위**: 캘리포니아 거주자의 개인정보를 수집하며 수익, 데이터 규모, 또는 데이터 판매 기준을 충족하는 사업체에 적용됩니다.

**주요 의무**:
- **알 권리**: 소비자가 수집, 사용, 공유된 개인정보의 공개를 요청할 수 있음
- **삭제 권리**: 소비자가 개인정보 삭제를 요청할 수 있음
- **옵트아웃 권리**: 소비자가 개인정보의 판매 또는 공유를 거부할 수 있음
- **정정 권리**: 소비자가 부정확한 개인정보의 정정을 요청할 수 있음 (CPRA 추가)
- **민감한 개인정보 사용 제한 권리**: 소비자가 민감한 개인정보의 사용을 특정 목적으로 제한할 수 있음 (CPRA 추가)
- **차별 금지**: 권리를 행사한 소비자에 대해 차별할 수 없음
- **개인정보 처리방침**: 수집 시 또는 수집 전에 수집되는 개인정보 카테고리 및 목적을 설명하는 처리방침 제공 필수
- **서비스 제공자 계약**: 서비스 제공자와의 계약에서 개인정보 사용을 지정된 사업 목적으로 제한해야 함

**응답 기한**:
- 접수 후 10 영업일 이내 접수 확인
- 45 역일 이내 실질적 응답(통지 후 45일 연장 가능)

### 기타 주요 규제 모니터링

| 규제 | 관할권 | 주요 차별점 |
|---|---|---|
| **LGPD** (브라질) | 브라질 | GDPR과 유사; DPO 지정 필요; 국가 데이터 보호 당국(ANPD) 집행 |
| **POPIA** (남아프리카) | 남아프리카 | 정보 규제 기관 감독; 처리 등록 필요 |
| **PIPEDA** (캐나다) | 캐나다(연방) | 동의 기반 프레임워크; OPC 감독; 현대화 진행 중 |
| **PDPA** (싱가포르) | 싱가포르 | Do Not Call 레지스트리; 의무적 침해 통지; PDPC 집행 |
| **Privacy Act** (호주) | 호주 | 호주 프라이버시 원칙(APPs); 통지 가능 데이터 침해 제도 |
| **PIPL** (중국) | 중국 | 엄격한 국경 간 이전 규칙; 데이터 현지화 요건; CAC 감독 |
| **UK GDPR** | 영국 | 브렉시트 이후 영국 버전; ICO 감독; UK 특정 적정성과 함께 EU GDPR과 유사 |

## DPA 검토 체크리스트

데이터 처리 계약 또는 데이터 처리 부록을 검토할 때 다음을 확인합니다:

### 필수 요소 (GDPR 제28조)

- [ ] **처리 대상 및 기간**: 처리 범위와 기간이 명확히 정의됨
- [ ] **성격 및 목적**: 어떤 처리가 발생하고 왜 발생하는지 구체적 설명
- [ ] **개인정보 유형**: 처리되는 개인정보의 카테고리
- [ ] **정보주체 카테고리**: 누구의 개인정보가 처리되는지
- [ ] **컨트롤러의 의무 및 권리**: 컨트롤러의 지시 및 감독 권한

### 프로세서 의무

- [ ] **문서화된 지시에 따른 처리만**: 프로세서가 컨트롤러의 지시에 따라서만 처리할 것을 약속(법적 요건에 의한 예외 포함)
- [ ] **기밀성**: 처리 권한을 가진 직원이 기밀 유지를 약속
- [ ] **보안 조치**: 적절한 기술적·조직적 조치 기술(제32조 참조)
- [ ] **하위 프로세서 요건**:
  - [ ] 서면 승인 요건(일반 또는 특정)
  - [ ] 일반 승인인 경우: 이의 제기 기회와 함께 변경 통지
  - [ ] 하위 프로세서가 서면 계약을 통해 동일한 의무에 구속
  - [ ] 프로세서가 하위 프로세서의 이행에 대해 계속 책임
- [ ] **정보주체 권리 지원**: 프로세서가 정보주체 요청 대응에서 컨트롤러를 지원
- [ ] **보안 및 침해 지원**: 프로세서가 보안 의무, 침해 통지, DPIA, 사전 협의에서 지원
- [ ] **삭제 또는 반환**: 종료 시 모든 개인정보를 삭제 또는 반환(컨트롤러 선택에 따라)하고 법적 보관이 필요한 경우를 제외하고 기존 사본 삭제
- [ ] **감사 권한**: 컨트롤러가 감사 및 검사를 수행할 권한(또는 제3자 감사 보고서 수락)
- [ ] **침해 통지**: 프로세서가 개인정보 침해를 지체 없이 컨트롤러에 통지(이상적으로 24-48시간 이내; 컨트롤러가 72시간 규제 기한을 준수할 수 있도록 해야 함)

### 국제 이전

- [ ] **이전 메커니즘 식별**: SCCs, 적정성 결정, BCRs, 또는 기타 유효한 메커니즘
- [ ] **SCCs 버전**: 해당하는 경우 현행 EU SCCs(2021년 6월 버전) 사용
- [ ] **올바른 모듈**: 적절한 SCC 모듈 선택(C2P, C2C, P2P, P2C)
- [ ] **이전 영향 평가**: 적정성 결정이 없는 국가로의 이전 시 완료
- [ ] **보충 조치**: 이전 영향 평가에서 식별된 공백을 해결하기 위한 기술적, 조직적, 계약적 조치
- [ ] **UK 부록**: 영국 개인정보가 범위에 포함된 경우 UK International Data Transfer Addendum 포함

### 실무적 고려사항

- [ ] **책임**: DPA 책임 조항이 주요 서비스 계약과 일치(또는 상충하지 않음)
- [ ] **종료 정합성**: DPA 기간이 서비스 계약과 일치
- [ ] **데이터 위치**: 처리 위치가 명시되고 수용 가능
- [ ] **보안 표준**: 특정 보안 표준 또는 인증 요구(SOC 2, ISO 27001 등)
- [ ] **보험**: 데이터 처리 활동에 대한 적절한 보험 적용

### 일반적인 DPA 이슈

| 이슈 | 위험 | 표준 입장 |
|---|---|---|
| 통지 없는 포괄적 하위 프로세서 승인 | 처리 체인에 대한 통제력 상실 | 이의 제기 권한과 함께 통지 요구 |
| 침해 통지 기한 > 72시간 | 적시 규제 통지 불가능 | 24-48시간 이내 통지 요구 |
| 감사 권한 없음(또는 제3자 보고서를 통한 감사 권한만) | 준수 확인 불가 | SOC 2 Type II 수용 + 사유 발생 시 감사 권한 |
| 데이터 삭제 기한 미명시 | 데이터 무기한 보관 | 종료 후 30-90일 이내 삭제 요구 |
| 데이터 처리 위치 미명시 | 데이터가 어디서든 처리될 수 있음 | 처리 위치 공개 요구 |
| 구 버전 SCCs | 유효하지 않은 이전 메커니즘 | 현행 EU SCCs(2021년 버전) 요구 |

## 정보주체 요청 처리

### 요청 접수

정보주체 요청을 수신한 경우:

1. **요청 유형 식별**:
   - 접근(개인정보 사본)
   - 정정(부정확한 데이터 수정)
   - 삭제("잊혀질 권리")
   - 처리 제한
   - 데이터 이동성(구조화된, 기계 판독 가능한 형식)
   - 처리에 대한 이의 제기
   - 판매/공유 옵트아웃(CCPA/CPRA)
   - 민감한 개인정보 사용 제한(CPRA)

2. **적용 가능한 규제 식별**:
   - 정보주체의 소재지는?
   - 조직의 존재 및 활동에 기반하여 어떤 법률이 적용되는가?
   - 구체적 요건 및 기한은?

3. **신원 확인**:
   - 요청자가 주장하는 본인인지 확인
   - 데이터의 민감성에 비례하는 합리적 확인 조치 사용
   - 과도한 문서를 요구하지 않음

4. **요청 기록**:
   - 수신일
   - 요청 유형
   - 요청자 신원
   - 적용 규제
   - 응답 기한
   - 담당자

### 응답 기한

| 규제 | 최초 접수 확인 | 실질적 응답 | 연장 |
|---|---|---|---|
| GDPR | 미지정(모범 사례: 신속히) | 30일 | +60일(통지 포함) |
| CCPA/CPRA | 10 영업일 | 45 역일 | +45일(통지 포함) |
| UK GDPR | 미지정(모범 사례: 신속히) | 30일 | +60일(통지 포함) |
| LGPD | 미지정 | 15일 | 제한적 연장 |

### 면제 및 예외

요청 이행 전 면제 적용 여부를 확인합니다:

**규제 전반의 일반적 면제**:
- 법적 청구의 방어 또는 성립
- 보관을 요구하는 법적 의무
- 공익 또는 공적 권한
- 표현의 자유 및 정보(삭제 요청)
- 공익 목적 또는 과학적/역사적 연구를 위한 보관

**조직별 고려사항**:
- 소송 보전: 법적 보전 대상 데이터는 삭제 불가
- 규제적 보관: 재무 기록, 고용 기록, 기타 카테고리에 의무적 보관 기간이 있을 수 있음
- 제3자 권리: 요청 이행이 타인의 권리에 부정적 영향을 미칠 수 있음

### 응답 프로세스

1. 시스템 전반에서 요청자의 모든 개인정보 수집
2. 면제 적용 및 근거 문서화
3. 응답 준비: 요청 이행 또는 이행 불가 사유(전체 또는 부분) 설명
4. 거부(전체 또는 부분)하는 경우: 거부의 구체적 법적 근거 인용
5. 감독 당국에 불만을 제기할 권리를 요청자에게 고지
