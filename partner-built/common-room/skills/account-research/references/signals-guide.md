# Common Room 신호 유형 - 해석 가이드

## 신호 범주

### 제품 신호(1st-party)

| Signal | What it means |
|--------|--------------|
| Active seats increasing | 계정 내 확장 - 아웃리치 기회 |
| Feature adoption (new features used) | 제품 참여도 심화 - 확장 대화에 좋음 |
| Login frequency spike | 참여도 증가 - 이유를 알아보기 위해 후속 확인 필요 |
| Login frequency drop | 참여 저하 또는 이탈 위험 - 선제적 아웃리치 필요 |
| Trial started | 적극적 평가 - 시간 민감, 우선순위 높음 |
| Trial nearing expiry | 의사결정 시점이 다가옴 - 즉시 대응 |
| Support tickets opened | 잠재적 마찰 - 패턴 확인 |

### 커뮤니티 신호(2nd-party)

| Signal | What it means |
|--------|--------------|
| New community post | 적극적으로 참여 중, 질문이나 아이디어가 있음 |
| Reply / reaction to post | 커뮤니티 참여 |
| Forum thread started | 도움을 구하거나 피드백을 공유 중 - 직접 참여 가능 |
| Documentation viewed (specific page) | 특정 기능을 조사 중 - 확장 영역과 연결되면 가치 높음 |
| Event registration/attendance | 강한 참여 및 의도 신호 |

### 의도 신호(3rd-party)

| Signal | What it means |
|--------|--------------|
| Hiring for relevant roles | 팀이 성장 중, 예산 존재 |
| Funding announced | 새 예산이 생겼을 가능성 높음 - 축하 아웃리치 |
| Company news / press mention | 아웃리치 훅으로 좋음 |
| Tech stack change | 새 통합이 필요할 수 있음 |
| Competitive product removed from stack | 기회 창구 - 매우 높은 우선순위 |
| Website activity spike | 적극적 조사 단계 |

### 관계 신호

| Signal | What it means |
|--------|--------------|
| Recent meeting logged | 활성 관계 - 맥락이 있음 |
| No meeting in 90+ days | 관계가 식고 있음 - 체크인 필요 |
| Open opportunity in CRM | 활성 딜 - AE와 조율 |
| Champion job change | 관계 리스크 또는 기회 |

## 신호 해석 원칙

1. **최신성이 가장 중요합니다.** 어제의 신호는 6개월 전 신호보다 훨씬 강합니다. 항상 타임스탬프를 적으세요.
2. **단일 신호보다 신호 묶음이 더 중요합니다.** 같은 방향의 여러 신호는 하나의 신호보다 훨씬 강합니다.
3. **신호 부재도 신호입니다.** 유료 계정에서 사용량이 적고 커뮤니티 참여도 없다면 위험 신호입니다.
4. **계정 단계에 맞춰 해석하세요.** trial에서는 전환 신호가 중요합니다. 성숙 고객에서는 확장과 이탈 위험이 중요합니다. prospect에서는 의도 신호가 가장 중요합니다.
