# 플랫폼별 검색 전략

브랜드 자료 탐색 범위를 극대화하기 위한 플랫폼별 쿼리 패턴과 팁입니다.

## Notion (주요 탐색 엔진)

Notion AI Search는 연결된 소스(Google Drive, SharePoint, OneDrive, Slack, Jira, Teams)를 통합 검색하므로, 가장 가치 있는 단일 검색 엔드포인트입니다.

### 쿼리 패턴

**직접 브랜드 검색:**
- "brand guidelines"
- "style guide"
- "brand voice"
- "tone of voice"
- "messaging framework"

**영업 및 마케팅 콘텐츠:**
- "pitch deck"
- "sales playbook"
- "email templates"
- "value proposition"
- "competitive positioning"

**운영 브랜드 콘텐츠:**
- "brand update"
- "rebrand"
- "launch messaging"
- "press release template"
- "social media guidelines"

### 팁
- Notion AI Search는 연결된 소스의 결과를 반환합니다 — 한 번의 검색으로 여러 플랫폼을 커버
- 회사 이름 + "brand"로 검색하여 회사별 가이드라인 찾기
- "Brand Assets", "Marketing Resources", "Content Library"라는 제목의 데이터베이스 확인
- "brand", "marketing", "style"로 태그된 페이지 찾기

## Atlassian Confluence

기업에서는 공식 브랜드 문서를 Confluence 스페이스에 자주 저장합니다.

### 쿼리 패턴

**스페이스 수준 검색:**
- "Marketing", "Brand", "Communications", "Sales Enablement"라는 이름의 스페이스 검색
- "brand", "style-guide", "guidelines" 레이블이 있는 스페이스 확인

**페이지 수준 검색:**
- "brand style guide"
- "voice and tone guidelines"
- "messaging framework"
- "content standards"
- "editorial guidelines"

**템플릿 검색:**
- "email template"
- "proposal template"
- "presentation template"

### 팁
- Confluence 스페이스는 종종 계층적 페이지 트리를 가집니다 — 브랜드 루트 페이지를 찾고 하위 페이지 탐색
- 최근 업데이트된 페이지 확인 (브랜드 가이드라인은 진화합니다)
- 구독자가 많은 페이지 찾기 — 중요한 공유 콘텐츠를 나타냄
- Confluence 페이지에 업로드된 PDF 브랜드 가이드의 첨부 파일 검색

## Box

클라우드 파일 저장소 — 공식 브랜드 문서, 공유 자료, 스타일 가이드가 자주 보관됩니다.

### 쿼리 패턴

**폴더 검색:**
- "Brand Guidelines"
- "Marketing Assets"
- "Brand Kit"
- "Style Guide"
- "Sales Collateral"

**문서 검색:**
- "brand guide" (PDF, DOCX, Word)
- "style guide" (PDF, DOCX, Word)
- "messaging document"
- "brand standards"

### 팁
- Box에는 종종 PDF나 Word 문서로 된 "진실의 근원(source of truth)" 브랜드 가이드가 포함됨
- 회사 전체 접근이 가능한 공유 폴더 확인
- 마케팅 또는 브랜드 팀과 공유된 폴더 확인
- 최근 수정된 브랜드 문서를 검색하여 최신 버전 찾기
- Box 메타데이터 검색을 사용하여 콘텐츠 유형이나 사용자 정의 속성으로 필터링

## Google Drive

Google Drive에는 공유 브랜드 문서, 마케팅 자료, 공식 스타일 가이드가 저장됩니다.

### 쿼리 패턴

**폴더 검색:**
- "Brand Guidelines"
- "Marketing"
- "Brand Assets"
- "Style Guide"

**문서 검색:**
- "brand guide" (PDF, Google Docs, Google Slides)
- "style guide"
- "messaging framework"
- "brand standards"
- "brand voice"

### 팁
- 공유 드라이브 확인 — 브랜드 자료는 종종 팀 전체 공유 드라이브에 있음
- 최근 수정된 브랜드 문서를 검색하여 최신 버전 찾기
- 소유자(마케팅 팀원)별 검색으로 브랜드 소유 콘텐츠 찾기
- Google Docs와 Slides는 살아있는 브랜드 문서의 일반적인 형식
- 브랜드 개요 자료가 포함된 Google Slides 프레젠테이션 확인

## Microsoft 365 (SharePoint / OneDrive)

기업 조직에서는 종종 SharePoint 사이트에 브랜드 문서를 집중화합니다.

### 쿼리 패턴

**SharePoint 사이트 검색:**
- 마케팅 또는 커뮤니케이션 SharePoint 사이트 검색
- "brand guidelines"
- "style guide"
- "brand standards"
- "messaging framework"

**문서 라이브러리 검색:**
- 마케팅/커뮤니케이션 사이트의 문서 라이브러리 확인
- "brand guide" (Word, PDF, PowerPoint)
- "brand book"
- "editorial guidelines"

**OneDrive 검색:**
- 공유 OneDrive 폴더에서 "brand" 파일
- 조직과 공유된 "style guide"

### 팁
- 먼저 마케팅/커뮤니케이션 SharePoint 사이트 확인 — 브랜드 문서의 가장 일반적인 위치
- PDF 브랜드 가이드와 PowerPoint 브랜드 자료가 있는 문서 라이브러리 검색
- SharePoint 메타데이터를 사용하여 브랜드 태그 콘텐츠 찾기
- OneDrive 공유 폴더에 브랜드 자료의 작업 초안이 포함될 수 있음
- 사용 가능한 경우 SharePoint 검색 및 문서 라이브러리 도구 사용

## Slack

Slack 채널에는 비공식 브랜드 토론, 결정 사항, 발전하는 브랜드 보이스 패턴이 포함됩니다.

### 쿼리 패턴

**채널 검색:**
- 채널 찾기: #brand, #marketing, #brand-voice, #style-guide, #creative
- 브랜드 관련 콘텐츠가 있는 채널 주제 및 설명 확인

**메시지 검색:**
- "brand guidelines"
- "brand voice"
- "tone of voice"
- "style guide"
- "brand update"

**고정 메시지:**
- #brand 및 #marketing 채널의 고정 메시지 확인
- 고정 항목에는 종종 승인된 브랜드 리소스나 결정 사항이 포함됨

### 팁
- 먼저 #brand 및 #marketing 채널 검색 — 브랜드 토론이 있을 가능성이 가장 높은 곳
- 고정 메시지 확인 — 팀에서 종종 브랜드 가이드라인과 결정 사항을 고정
- 브랜드 진화에 대한 맥락을 위한 브랜드 토론 스레드 찾기
- Slack은 대화 소스입니다 — 소스 분류에서 CONVERSATIONAL 등급
- 최근 메시지에 아직 공식적으로 문서화되지 않은 브랜드 변경 사항이 있을 수 있음

## Gong (대화 인텔리전스)

영업 통화 트랜스크립트는 브랜드가 실제로 어떻게 커뮤니케이션하는지 보여줍니다.

### 쿼리 패턴

**통화 검색:**
- "won", "closed-won" 태그 통화 검색 (성공적인 브랜드 적용)
- 최고 성과자별 필터링 (그들의 언어 패턴이 암묵적 브랜드 보이스를 정의)
- "demo", "discovery", "closing" 태그 통화 찾기

**트랜스크립트 검색:**
- 회사별 가치 제안이 포함된 트랜스크립트 검색
- 성공적인 통화에서 반복되는 표현 찾기
- 경쟁사가 언급된 통화 찾기 (포지셔닝 파악)

### 팁
- 성공적인 통화에 집중 — 효과가 있는 브랜드 보이스를 대표
- 최고 성과자의 언어를 평균 성과자의 언어와 비교
- 일관된 오프닝 라인과 클로징 패턴 찾기
- 이의 처리 언어 기록 — 압박 상황에서의 브랜드 포지셔닝 파악

## Granola (회의 인텔리전스)

AI 회의 메모장의 회의록과 트랜스크립트입니다.

### 쿼리 패턴

**회의 검색:**
- "brand", "positioning", "messaging" 관련 회의 쿼리
- 최근 회의 목록을 나열하고 고객 대면 통화 필터링
- 전략 세션 및 브랜드 기획 회의 검색

**트랜스크립트 조회:**
- 높은 신호 회의의 전체 트랜스크립트 조회
- 회의록에서 반복 주제 찾기
- 핵심 이해관계자(마케팅 리드, 임원)와의 회의 찾기

### 팁
- Granola는 회의록과 트랜스크립트 모두 캡처 — 둘 다 유용
- 회의록에는 종종 요약된 브랜드 결정 사항과 실행 항목이 포함됨
- 반복 회의 시리즈 찾기 (주간 브랜드 싱크, 마케팅 스탠드업)
- 두 가지 모두 사용 가능한 경우 Granola 회의록과 Gong 통화 트랜스크립트 교차 참조

## Figma (브랜드 디자인 시스템)

시각적 브랜드 요소는 보이스와 톤에 간접적으로 영향을 줍니다.

### 쿼리 패턴

**파일 검색:**
- "brand design system"
- "design tokens"
- "brand kit"
- "style guide"
- "component library"

**컴포넌트 검색:**
- 브랜드 색상, 타이포그래피 스케일이 있는 파일 찾기
- 사용 가이드라인이 포함된 컴포넌트 문서 찾기
- 디자인 시스템 문서에서 콘텐츠/카피 가이드라인 확인

### 팁
- Figma 디자인 시스템에는 시각적 사양과 함께 작성 가이드라인이 포함되는 경우가 많음
- 브랜드 맥락을 위한 파일 설명과 코멘트 확인
- 디자인 시스템 문서 내 "voice and tone" 섹션 찾기
- 디자인 파일의 브랜드 성격 설명이 보이스 속성을 보여줌

## 크로스 플랫폼 검색 팁

### 커버리지 극대화
1. Notion AI Search로 시작 (통합을 통한 가장 넓은 커버리지)
2. 기업 문서를 위한 Confluence 후속 검색
3. 공식 브랜드 문서를 위한 Google Drive 확인
4. 클라우드 저장 브랜드 문서를 위한 Box 확인
5. 기업 파일을 위한 SharePoint/OneDrive 검색
6. 브랜드 토론과 결정 사항을 위한 Slack 검색
7. 대화 브랜드 패턴을 위한 Gong 검색
8. 회의 트랜스크립트와 메모를 위한 Granola 검색
9. 디자인에 내장된 브랜드 가이드라인을 위한 Figma 검토

### 중복 방지
- 소스 URL을 추적하여 여러 플랫폼에서 동일 문서 감지
- 같은 콘텐츠가 여러 플랫폼에 있을 때 가장 최근 업데이트된 버전 선호
- Notion 통합이 다른 플랫폼의 결과를 반환할 때 이중 검색을 피하기 위해 기록

### 최신성 집중
- 운영, 대화, 맥락 소스의 경우 최근 12개월 콘텐츠에 집중
- 명시적 브랜드 문서(스타일 가이드, 브랜드 북)를 찾을 때만 더 이전 검색
- 결과에 오래된 콘텐츠가 포함되면 가장 최근 업데이트된 버전 선호
- 12개월 이상 된 비AUTHORITATIVE 소스 표시 — 구버전 포지셔닝을 반영할 수 있음

### 결과 없음 처리
- 플랫폼에서 결과가 없으면 더 넓은 쿼리 시도 ("marketing", "sales")
- 플랫폼이 실제로 연결되고 인증되었는지 확인
- 탐색 보고서에 공백 기록 — 플랫폼에 콘텐츠가 없다는 것 자체가 유용한 정보
