# /sprint-planning

> 익숙하지 않은 자리표시자가 보이거나 어떤 도구가 연결되어 있는지 확인해야 한다면 [CONNECTORS.md](../../CONNECTORS.md)를 참고하세요.

작업 범위를 정하고, 용량을 추정하고, 명확한 목표를 설정해 스프린트를 계획합니다.

## 사용법

```
/sprint-planning $ARGUMENTS
```

## 작동 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                    스프린트 계획                                  │
├─────────────────────────────────────────────────────────────────┤
│  독립 실행(항상 작동)                                             │
│  ✓ 스프린트 목표와 성공 기준 정의                                │
│  ✓ 팀 용량 추정(PTO, 회의 반영)                                  │
│  ✓ 백로그 항목 범위 설정과 우선순위 지정                         │
│  ✓ 의존성과 위험 식별                                            │
│  ✓ 스프린트 계획 문서 생성                                       │
├─────────────────────────────────────────────────────────────────┤
│  강화 모드(도구를 연결하면)                                       │
│  + 프로젝트 추적기: 백로그 가져오기, 스프린트 생성, 항목 배정   │
│  + 캘린더: 용량에 PTO와 회의 반영                                │
│  + 채팅: 스프린트 계획을 팀과 공유                               │
└─────────────────────────────────────────────────────────────────┘
```

## 필요한 정보

- **팀**: 이번 스프린트에 누가 있고, 각자의 가용성은 어떤가요?
- **스프린트 기간**: 며칠/몇 주인가요?
- **백로그**: 우선순위가 어떻게 되나요? (추적기에서 가져오거나, 붙여넣거나, 설명)
- **이월 작업**: 지난 스프린트에서 끝나지 않은 것이 있나요?
- **의존성**: 다른 팀에 막힌 것이 있나요?

## 출력

```markdown
## 스프린트 계획: [스프린트 이름]
**기간:** [시작] — [종료] | **팀:** [X]명 엔지니어
**스프린트 목표:** [성공이 어떤 모습인지 한 문장]

### 용량
| 사람 | 사용 가능 일수 | 배분 | 메모 |
|--------|---------------|------------|-------|
| [이름] | [Y]일 중 [X]일 | [X] 포인트/시간 | [PTO, 온콜 등] |
| **합계** | **[X]** | **[X] 포인트** | |

### 스프린트 백로그
| 우선순위 | 항목 | 추정치 | 담당자 | 의존성 |
|----------|------|----------|-------|--------------|
| P0 | [반드시 배포] | [X] pts | [사람] | [없음 / X에 의해 차단] |
| P1 | [배포해야 함] | [X] pts | [사람] | [없음] |
| P2 | [스트레치] | [X] pts | [사람] | [없음] |

### 계획 용량: [X] 포인트 | 스프린트 부하: [X] 포인트([X]%의 용량)

### 위험
| 위험 | 영향 | 완화책 |
|------|--------|------------|
| [위험] | [무슨 일이 생기는지] | [어떻게 할지] |

### 완료 기준
- [ ] 코드 리뷰 및 병합 완료
- [ ] 테스트 통과
- [ ] 문서 업데이트(해당 시)
- [ ] 제품 승인

### 주요 일정
| 날짜 | 이벤트 |
|------|-------|
| [날짜] | 스프린트 시작 |
| [날짜] | 중간 점검 |
| [날짜] | 스프린트 종료 / 데모 |
| [날짜] | 회고 |
```

## 팁

1. **여유를 남기세요** — 70-80% 용량으로 계획하세요. 중간에 끼어드는 일이 생깁니다.
2. **명확한 스프린트 목표 하나** — 한 문장으로 설명할 수 없다면 스프린트가 초점이 흐립니다.
3. **스트레치 항목을 식별하세요** — 예상보다 오래 걸리면 무엇을 자를지 알아두세요.
4. **솔직하게 이월하세요** — 무엇인가 배포되지 않았다면, 다시 약속하기 전에 왜 그랬는지 이해하세요.
