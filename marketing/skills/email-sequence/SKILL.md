# 이메일 시퀀스

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인해야 하는 경우, [CONNECTORS.md](../../CONNECTORS.md)를 참조하세요.

모든 라이프사이클 또는 캠페인 사용 사례에 대해 전체 카피, 타이밍, 분기 로직, 성과 벤치마크가 포함된 완전한 이메일 시퀀스를 설계하고 초안합니다.

## 트리거

사용자가 `/email-sequence`를 실행하거나 이메일 시퀀스, 드립 캠페인, 육성 플로우, 온보딩 시리즈를 생성, 설계, 구축 또는 초안하도록 요청합니다.

## 입력

사용자로부터 다음을 수집합니다. 제공되지 않으면 진행하기 전에 질문합니다:

1. **시퀀스 유형** — 다음 중 하나:
   - 온보딩
   - 리드 육성
   - 재참여
   - 제품 출시
   - 이벤트 후속
   - 업그레이드/업셀
   - 윈백
   - 교육 드립

2. **목표** — 시퀀스가 달성해야 할 것 (예: 새 사용자 활성화, 리드를 고객으로 전환, 이탈 감소, 이벤트 참석 유도, 더 높은 등급으로 업셀)

3. **대상** — 이 시퀀스를 받는 사람, 현재 단계, 관련 세분화 세부 정보 (역할, 산업, 행동 트리거, 라이프사이클 단계)

4. **이메일 수** (선택 사항) — 지정되지 않으면 아래 시퀀스 유형 템플릿 섹션의 템플릿을 기반으로 수를 추천

5. **타이밍/빈도 선호** (선택 사항) — 이메일 간 원하는 간격 (예: "3일마다", "주간", "첫 주 공격적 후 점진적 감소")

6. **브랜드 보이스** — 로컬 설정에 구성되어 있으면 자동 적용하고 사용자에게 알립니다. 구성되어 있지 않으면 질문합니다: "따를 브랜드 보이스 가이드라인이 있으신가요? 없으면 명확하고 대화적인 전문 톤을 사용하겠습니다."

7. **추가 맥락** (선택 사항):
   - 포함할 특정 제안, 할인 또는 인센티브
   - 링크할 CTA 또는 랜딩 페이지
   - 이용 가능한 콘텐츠 자산 (블로그 포스트, 사례 연구, 비디오, 가이드)
   - 강조할 제품 기능
   - 참조할 경쟁사 차별화 요소

## 프로세스

### 1. 시퀀스 전략

이메일을 초안하기 전에 전체 시퀀스 아키텍처를 정의합니다:

- **내러티브 아크** — 이 시퀀스가 모든 이메일에 걸쳐 어떤 이야기를 전하는가? 첫 이메일에서 마지막까지의 감정적, 논리적 진행은 무엇인가?
- **여정 매핑** — 각 이메일을 구매자 또는 사용자 여정의 단계에 매핑 (인지, 고려, 결정, 활성화, 확장)
- **에스컬레이션 로직** — 각 이메일의 강도, 긴급성, 가치가 이전 것을 어떻게 기반으로 구축되는가?
- **성공 정의** — 시퀀스가 역할을 다했고 수신자가 퇴장해야 한다는 것을 나타내는 구체적인 행동은 무엇인가?

### 2. 개별 이메일 설계

시퀀스의 각 이메일에 대해 제작합니다:

#### 제목줄
- 이메일당 2-3개 옵션 제공
- 접근 방식 다양화: 호기심, 혜택 중심, 긴급성, 개인화, 질문 기반
- 가능한 50자 이내 유지; 모바일에서의 미리보기 동작 참고

#### 미리보기 텍스트
- 제목줄을 보완 (반복하지 않는) 40-90자
- 오픈 가능성을 높이는 맥락이나 흥미를 추가해야 함

#### 이메일 목적
- 이 이메일이 왜 존재하고 수신자를 무엇을 향해 움직이는지 설명하는 한 문장

#### 본문 카피
- 바로 사용할 수 있는 전체 초안
- 명확한 계층: 후크, 본문, CTA
- 짧은 문단 (최대 2-3문장)
- 적절한 곳에 핵심 구문 굵게 처리한 스캔 가능한 서식
- 관련 있는 곳에 개인화 토큰 (예: 이름, 회사명, 사용 제품)

#### 주요 CTA
- 버튼 텍스트 및 목적지
- 이메일당 하나의 주요 CTA (시퀀스 단계에 적합한 경우에만 보조 CTA)

#### 타이밍
- 트리거 이벤트 또는 이전 이메일 이후 일수
- 참여에 따라 타이밍을 조정해야 하는 경우 참고 (예: "열었지만 클릭하지 않은 경우 더 빨리 발송")

#### 세그먼트/조건 참고
- 이 이메일을 받는 사람 vs. 건너뛰는 사람
- 행동 또는 속성 기반 조건 (예: "설정을 완료하지 않은 사용자에게만 발송")

### 3. 시퀀스 로직

시퀀스의 플로우 제어를 정의합니다:

- **분기 조건** — 참여에 따른 대체 경로. 예:
  - "이메일 2를 열었지만 CTA를 클릭하지 않은 경우, 이메일 3 대신 이메일 2b (더 부드러운 재요청) 발송"
  - "이메일 1에서 CTA를 클릭한 경우, 이메일 2를 건너뛰고 바로 이메일 3으로 이동"
- **종료 조건** — 수신자가 전환 (원하는 행동 완료)하면 시퀀스에서 제거합니다. 이 시퀀스에서 "전환"의 의미를 정의합니다.
- **재진입 규칙** — 누군가가 시퀀스에 다시 진입할 수 있는가? 어떤 조건에서? (예: "사용자가 90일 후 다시 이탈하면 윈백 시퀀스에 재진입")
- **억제 규칙** — 수신자가 이미 다른 활성 시퀀스에 있거나, 마케팅 수신을 거부했거나, 지난 48시간 내에 지원팀에 연락한 경우 발송하지 않음

### 4. 성과 벤치마크

사용자가 목표를 설정할 수 있도록 시퀀스 유형에 따른 예상 벤치마크를 제공합니다:

| 지표 | 온보딩 | 리드 육성 | 재참여 | 윈백 |
|--------|-----------|--------------|---------------|----------|
| 오픈율 | 50-70% | 20-30% | 15-25% | 15-20% |
| 클릭률 | 10-20% | 3-7% | 2-5% | 2-4% |
| 전환율 | 15-30% | 2-5% | 3-8% | 1-3% |
| 수신 거부율 | <0.5% | <0.5% | 1-2% | 1-3% |

사용자가 해당 맥락을 제공한 경우 산업 및 대상에 따라 벤치마크를 조정합니다.

## 시퀀스 유형 템플릿

시작 프레임워크로 사용합니다. 사용자의 목표와 대상에 따라 길이와 내용을 조정합니다.

**온보딩 (14-21일에 걸쳐 5-7개 이메일):**
환영 및 기대치 설정 -- 가치를 보여주는 빠른 성과 -- 핵심 기능 심층 소개 -- 고급 기능 또는 연동 -- 소셜 증거 및 커뮤니티 -- 확인 및 피드백 요청 -- 업그레이드 안내 또는 다음 단계

**리드 육성 (3-4주에 걸쳐 4-6개 이메일):**
가치 우선 교육 콘텐츠 -- 페인 포인트 식별 -- 증거를 통한 솔루션 포지셔닝 -- 소셜 증거 및 결과 -- 소프트 CTA (트라이얼, 데모, 리소스) -- 직접 CTA (구매, 예약, 가입)

**재참여 (10-14일에 걸쳐 3-4개 이메일):**
"보고 싶었습니다" + 돌아올 설득력 있는 이유 -- 놓치고 있는 것을 강조하는 가치 리마인더 -- 인센티브 또는 독점 제안 -- 명확한 마감일이 있는 마지막 기회

**윈백 (30일에 걸쳐 3-5개 이메일):**
무엇이 잘못되었는지 묻는 친절한 확인 -- 떠난 이후 새로워진 것 -- 복귀를 위한 특별 제안 또는 인센티브 -- 피드백 요청 (돌아오지 않더라도) -- 문을 열어둔 마지막 인사

**제품 출시 (2-3주에 걸쳐 4-6개 이메일):**
티저 또는 사전 발표 -- 전체 세부 사항이 포함된 출시 발표 -- 기능 스포트라이트 또는 사용 사례 -- 소셜 증거 및 초기 결과 -- 기간 한정 제안 또는 보너스 -- 마지막 기회 또는 리마인더

**이벤트 후속 (7-10일에 걸쳐 3-4개 이메일):**
핵심 시사점 또는 녹화와 함께 감사 -- 이벤트 리소스 모음 -- 관련 제안 또는 다음 단계 -- 피드백 설문

**업그레이드/업셀 (2-3주에 걸쳐 3-5개 이메일):**
사용 마일스톤 또는 성공 축하 -- 직면하고 있는 기능 갭 또는 제한 -- 증거를 통한 업그레이드 혜택 -- 기간 한정 인센티브 -- 플랜 직접 비교

**교육 드립 (4-6주에 걸쳐 5-8개 이메일):**
소개 및 배울 내용 -- 레슨 1: 기초 개념 -- 레슨 2: 중급 개념 -- 레슨 3: 고급 개념 -- 실습 적용 또는 연습 -- 리소스 모음 -- 수료 및 다음 단계

## 도구 연동

### ~~이메일 마케팅이 연결된 경우 (예: Klaviyo, Mailchimp, Customer.io)
- 플랫폼에서 시퀀스를 플로우 또는 자동화로 설정하는 방법 참조
- 사용할 플랫폼별 기능 참고 (예: 스마트 발송 시간, 조건부 분할, A/B 테스트)
- 분기 로직을 플랫폼의 비주얼 플로우 빌더 개념에 매핑

### ~~마케팅 자동화 또는 ~~CRM이 연결된 경우 (예: HubSpot, Marketo)
- 세분화 및 종료 조건에 참고할 리드 스코어링 데이터 참조
- 세그먼트별 메시징 맞춤화를 위해 라이프사이클 단계 데이터 사용
- CRM 속성 또는 리스트 멤버십 기반 등록 트리거 설정 방법 참고

### 도구가 연결되지 않은 경우
- 모든 이메일 콘텐츠를 복사-붙여넣기 가능한 형식으로 제공
- 모든 이메일 플랫폼에서 따를 수 있는 설정 체크리스트 포함:
  1. 자동화 또는 플로우 생성
  2. 등록 트리거 설정
  3. 지정된 딜레이로 각 이메일 추가
  4. 분기 및 종료 조건 구성
  5. 추천 지표에 대한 추적 설정

## 출력

다음 섹션이 포함된 완전한 시퀀스를 제시합니다:

### 시퀀스 개요 테이블

| # | 제목줄 | 목적 | 타이밍 | 주요 CTA | 조건 |
|---|-------------|---------|--------|-------------|-----------|

### 전체 이메일 초안
각 이메일에 제목줄 옵션, 미리보기 텍스트, 목적, 본문 카피, CTA, 타이밍, 세그먼트 참고가 포함됩니다.

### 시퀀스 플로우 다이어그램
이메일 플로우, 분기 경로, 종료 지점을 보여주는 텍스트 기반 다이어그램. 다음과 같은 명확한 형식을 사용합니다:

```
[트리거] --> 이메일 1 (0일차)
                |
          열었나? --네--> 이메일 2 (3일차)
                |              |
                아니오        CTA 클릭? --네--> [퇴장: 전환됨]
                |              |
                v              아니오
          이메일 1b (2일차)     |
                |              v
                +--------> 이메일 3 (7일차)
                               |
                               v
                          이메일 4 (10일차)
                               |
                          [퇴장: 시퀀스 완료]
```

### 분기 로직 참고
모든 조건, 퇴장, 억제를 참조 목록으로 요약합니다.

### A/B 테스트 제안
- 2-3개 추천 A/B 테스트 (제목줄, CTA 텍스트, 발송 시간, 이메일 길이)
- 무엇을 테스트할지, 어떻게 분할할지, 승자를 어떻게 측정할지

### 추적할 지표
- 시퀀스의 주요 전환 지표
- 이메일별 지표: 오픈율, CTR, 수신 거부율
- 시퀀스 수준 지표: 전체 전환율, 전환까지의 시간, 이탈 지점
- 추천 검토 주기 (예: "첫 달은 주간으로 성과를 검토하고, 이후 월간으로")

## 시퀀스 이후

질문합니다: "다음 중 어떤 것을 원하시나요:
- 특정 이메일의 카피나 톤 수정?
- 특정 시나리오에 대한 분기 경로 추가?
- 다른 대상 세그먼트를 위한 이 시퀀스의 변형 생성?
- 제목줄에 대한 A/B 테스트 변형 초안?
- 보완 시퀀스 구축 (예: 이 리드 육성이 전환된 후의 구매 후 후속)?"
