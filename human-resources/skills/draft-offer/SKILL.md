# /draft-offer

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인해야 한다면 [CONNECTORS.md](../../CONNECTORS.md)를 참고하세요.

신규 입사자를 위한 완전한 오퍼 레터를 작성합니다.

## 사용법

```
/draft-offer $ARGUMENTS
```

## 필요한 것

- **역할과 직함**: 어떤 포지션인가요?
- **레벨**: Junior, Mid, Senior, Staff 등
- **지역**: 어디에서 근무하나요? (보상과 복리후생에 영향)
- **보상**: 기본급, 지분, 입사 보너스(해당 시)
- **시작일**: 언제 시작하나요?
- **채용 관리자**: 누구에게 보고하나요?

모든 세부사항이 없어도, 함께 정리해 드리겠습니다.

## 출력

```markdown
## 오퍼 레터 초안: [Role] — [Level]

### 보상 패키지
| Component | Details |
|-----------|---------|
| **Base Salary** | $[X]/year |
| **Equity** | [X shares/units], [vesting schedule] |
| **Signing Bonus** | $[X] (if applicable) |
| **Target Bonus** | [X]% of base (if applicable) |
| **Total First-Year Comp** | $[X] |

### 조건
- **시작일**: [Date]
- **보고 대상**: [Manager]
- **근무지**: [Office / Remote / Hybrid]
- **고용 형태**: [Full-time, Exempt]

### 복리후생 요약
[후보자에게 중요한 복리후생 하이라이트]

### 오퍼 레터 본문

친애하는 [Candidate Name] 님,

저희는 [Company]의 [Title] 직책을 제안드리게 되어 기쁩니다...

[전체 오퍼 레터 본문]

### 채용 관리자 참고사항
- [필요 시 협상 가이드]
- [보상 밴드 맥락]
- [기타 주의사항 또는 고려사항]
```

## 연결 도구가 있을 경우

**~~HRIS**가 연결되어 있다면:
- 레벨/역할에 대한 보상 밴드 데이터를 가져옵니다
- 인력 승인 여부를 확인합니다
- 복리후생 세부사항을 자동으로 채웁니다

**~~ATS**가 연결되어 있다면:
- 지원서에서 후보자 세부사항을 가져옵니다
- 파이프라인의 오퍼 상태를 업데이트합니다

## 팁

1. **총보상을 포함하세요** - 후보자는 기본급만이 아니라 총보상을 비교합니다.
2. **지분을 구체적으로 작성하세요** - 주식 수, 현재 평가 방식, 베스팅 스케줄.
3. **개인화하세요** - 인터뷰 과정에서 나온 내용을 언급해 따뜻하게 만드세요.
