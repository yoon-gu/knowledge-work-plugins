# /people-report

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인해야 한다면 [CONNECTORS.md](../../CONNECTORS.md)를 참고하세요.

HR 데이터에서 인력 분석 보고서를 생성합니다. 인력 데이터를 분석해 추세, 위험, 기회를 드러냅니다.

## 사용법

```
/people-report $ARGUMENTS
```

## 보고 유형

**Headcount**: 현재 조직 스냅샷 - 팀, 지역, 레벨, 재직 기간별
**Attrition**: 이직 분석 - 자발/비자발, 팀별, 추세
**Diversity**: 대표성 지표 - 레벨, 팀, 파이프라인별
**Org Health**: 관리 범위, 관리 계층, 팀 규모, 이탈 위험

## 핵심 지표

### 유지율
- 전체 이직률(자발 + 비자발)
- 아쉬운 이직률
- 평균 재직 기간
- 이탈 위험 지표

### 다양성
- 레벨, 팀, 기능별 대표성
- 파이프라인 다양성(인구통계별 채용 퍼널)
- 그룹별 승진률
- 보상 형평성 분석

### 참여도
- 설문 점수와 추세
- eNPS(Employee Net Promoter Score)
- 참여율
- 서술형 피드백 주제

### 생산성
- 직원 1인당 매출
- 관리 범위 효율성
- 신규 입사자의 생산성 도달 시간

## 접근 방식

1. 상대가 어떤 질문에 답하려는지 이해합니다.
2. 적절한 데이터를 식별합니다(업로드, 붙여넣기, 또는 ~~HRIS에서 가져오기).
3. 적절한 통계 방법으로 분석합니다.
4. 맥락과 주의점을 함께 담아 결과를 제시합니다.
5. 데이터에 기반한 구체적인 조치를 권고합니다.

## 필요한 것

CSV를 업로드하거나 데이터를 설명해 주세요. 유용한 필드:
- 직원 이름/ID, 부서, 팀
- 직함, 레벨, 지역
- 시작일, 종료일(해당하는 경우)
- 관리자, 보상(관련이 있다면)
- 인구통계 정보(다양성 보고서용, 사용 가능하다면)

## 출력

```markdown
## 인력 보고서: [유형] — [날짜]

### 요약
[2-3개의 핵심 인사이트]

### 핵심 지표
| 지표 | 값 | 추세 |
|--------|-------|-------|
| [지표] | [값] | [상승/하락/유지] |

### 상세 분석
[특정 보고 유형에 대한 차트, 표, 서술]

### 권고 사항
- [데이터 기반 권고]
- [실행 항목]

### 방법론
[숫자를 계산한 방식과 주의사항]
```

## 연결 도구가 있을 경우

**~~HRIS**가 연결되어 있다면:
- 실시간 직원 데이터(인력 수, 재직 기간, 부서, 레벨)를 가져옵니다.
- CSV 업로드 없이 보고서를 생성합니다.

**~~chat**가 연결되어 있다면:
- 관련 채널에 보고서 요약을 공유할 수 있습니다.
