# scverse 품질 관리 가이드라인

이 문서는 scverse 생태계 권장 사항에 따른 단일 세포 RNA-seq 데이터의 품질 관리 모범 사례에 대한 자세한 정보를 제공합니다.

## 품질 관리 지표

### Count Depth (총 카운트)
- **측정 항목**: 세포당 총 UMI/reads 수
- **중요한 이유**: 낮은 카운트의 세포는 빈 드롭렛, 잔해 또는 포획이 불량한 세포일 수 있음
- **일반적 범위**: 세포당 500-50,000 카운트 (프로토콜에 따라 다름)
- **경고 신호**: 이중 분포(bimodal distribution)는 고품질과 저품질 세포의 혼합을 나타낼 수 있음

### Gene Detection (세포당 유전자 수)
- **측정 항목**: 최소 1 카운트 이상인 유전자 수
- **중요한 이유**: count depth와 강한 상관관계가 있으며, 낮은 값은 포획 불량을 나타냄
- **일반적 범위**: 세포당 200-5,000개 유전자
- **경고 신호**: 매우 낮은 값(<200)은 기술적 실패를 시사함

### 미토콘드리아 함량
- **측정 항목**: 미토콘드리아 유전자의 카운트 비율
- **중요한 이유**: 높은 MT%는 세포 스트레스, 세포자멸사(apoptosis) 또는 용해된 세포를 나타냄
- **일반적 범위**: 대부분의 조직에서 <5%, 대사 활성이 높은 세포의 경우 10-15%까지
- **종별 패턴**:
  - 마우스: 유전자가 'mt-'로 시작 (예: mt-Nd1, mt-Co1)
  - 인간: 유전자가 'MT-'로 시작 (예: MT-ND1, MT-CO1)
- **맥락이 중요함**: 일부 세포 유형(심근세포, 뉴런)은 자연적으로 더 높은 MT 함량을 가짐

### 리보솜 함량
- **측정 항목**: 리보솜 단백질 유전자의 카운트 비율
- **중요한 이유**: 세포 상태 또는 오염을 나타낼 수 있음
- **패턴**: 유전자가 'Rpl'/'RPL' (대소단위) 또는 'Rps'/'RPS' (소소단위)로 시작
- **참고**: 높은 리보솜 함량이 항상 나쁜 것은 아님 - 대사 활성이 높은 세포는 더 많은 리보솜을 가짐

### 헤모글로빈 함량
- **측정 항목**: 헤모글로빈 유전자의 카운트 비율
- **중요한 이유**: 비혈액 조직에서 혈액 오염을 나타냄
- **패턴**: '^Hb[^(p)]' 또는 '^HB[^(P)]'와 일치하는 유전자 (Hbp1/HBP1 제외)
- **사용 시기**: 조직 샘플(뇌, 간 등)에서 특히 중요

## MAD 기반 필터링 근거

### 왜 고정 임계값 대신 MAD를 사용하는가?

고정 임계값(예: "500개 미만의 유전자를 가진 세포 제거")은 다음과 같은 이유로 실패합니다:
- 다른 프로토콜은 다른 범위를 산출함
- 다른 조직은 다른 특성을 가짐
- 다른 종은 다른 유전자 수를 가짐
- 고정 임계값은 임의적이며 데이터 기반이 아님

MAD (Median Absolute Deviation, 중앙값 절대 편차)는 이상치에 강건하며 데이터셋에 적응합니다:
```
MAD = median(|X - median(X)|)
Outlier bounds = median ± n_MADs × MAD
```

### 권장 MAD 임계값

scverse 모범 사례를 따르며 (의도적으로 관대함):

**count depth에 대한 5 MADs (로그 변환)**
- 희귀 세포 집단을 유지하기 위해 매우 관대함
- 극단적 이상치(빈 드롭렛, 잔해)를 포착함
- 로그 변환은 일반적인 오른쪽 치우침 분포를 처리함

**유전자 수에 대한 5 MADs (로그 변환)**
- count depth 필터링과 병행
- count 필터링과 결합할 때 가장 유의미함
- 로그 변환이 분포를 정규화함

**미토콘드리아 비율에 대한 3 MADs**
- 높은 MT%가 죽어가는 세포를 강하게 나타내므로 더 엄격함
- 원시 비율 사용 (로그 변환하지 않음)
- 추가 엄격성을 위해 하드 임계값과 결합

**하드 임계값: 8% 미토콘드리아 함량**
- MAD 기반 탐지를 넘어서는 추가 필터
- 대부분의 조직에서 작동하는 보수적 기준점
- 대사 활성이 높은 세포 유형의 경우 더 높게 조정 (10-15%)

### 왜 관대하게 설정하는가?

기본 임계값은 의도적으로 세포를 유지하는 쪽으로 설정합니다. 그 이유는:
1. **희귀 집단**: 엄격한 필터링은 희귀하지만 생존 가능한 세포 유형을 제거할 수 있음
2. **생물학적 변이**: 일부 건강한 세포는 자연적으로 극단적인 값을 가짐
3. **가역성**: 나중에 더 많이 필터링하는 것이 손실된 세포를 복구하는 것보다 쉬움
4. **하류 분석의 강건성**: 최신 정규화 방법은 중간 수준의 품질 변이를 처리할 수 있음

## QC 시각화 해석

### 히스토그램
- **이중 분포**: 세포 유형의 혼합 또는 품질 문제를 나타낼 수 있음
- **긴 꼬리**: count depth에서 흔함; MAD 필터링이 이를 처리함
- **급격한 절단**: 사전 필터링 또는 기술적 아티팩트를 나타낼 수 있음

### 바이올린 플롯
- 분포 형태와 밀도를 보여줌
- 대칭 분포의 경우 중앙값(선)과 평균(다이아몬드)이 유사해야 함
- 넓은 분포는 높은 이질성을 시사함

### 산점도

**Counts vs Genes (MT%로 색상 표시)**
- 강한 양의 상관관계를 보여야 함 (일반적으로 R² > 0.8)
- 추세에서 벗어나는 점은 이상치일 수 있음
- 높은 MT% 세포는 종종 낮은 counts/genes에 클러스터링됨

**Counts vs MT%**
- 음의 상관관계 예상 (죽어가는 세포는 더 적은 카운트를 가짐)
- 수직 계층화는 배치 효과를 나타낼 수 있음
- 높은 counts + 높은 MT%인 세포는 의심스러움

**Genes vs MT%**
- counts vs MT%와 유사하나 종종 더 약한 상관관계
- 유전자 탐지 문제가 있는 세포를 식별하는 데 유용

## 유전자 필터링

세포 필터링 후, 20개 미만의 세포에서 탐지된 유전자를 제거합니다:
- **왜 20인가?**: 노이즈 감소와 정보 보존 사이의 균형
- **이점**: 데이터셋 크기 감소, 계산 속도 향상, 노이즈가 많은 유전자 제거
- **트레이드오프**: 매우 희귀한 마커를 잃을 수 있음; 희귀 집단을 연구하는 경우 10으로 조정

## 종별 고려사항

### 마우스 (Mus musculus)
- 미토콘드리아 유전자: mt-* (소문자)
- 리보솜 유전자: Rpl*, Rps* (첫 글자 대문자)
- 헤모글로빈 유전자: Hb* (단, Hbp1 제외)

### 인간 (Homo sapiens)
- 미토콘드리아 유전자: MT-* (대문자)
- 리보솜 유전자: RPL*, RPS* (모두 대문자)
- 헤모글로빈 유전자: HB* (단, HBP1 제외)

### 기타 종
스크립트에서 유전자 이름 패턴을 해당 생물체의 유전자 명명법에 맞게 조정하세요. 올바른 접두사를 확인하려면 Ensembl 또는 참조 주석을 참조하세요.

## 매개변수 조정 시기

다음과 같은 경우 필터링 임계값 조정을 고려하세요:

**더 엄격하게 (낮은 MADs)**
- 높은 주변 RNA 오염이 의심되는 경우
- 시각화에서 많은 저품질 세포가 관찰되는 경우
- 하류 분석에서 품질 기반 클러스터링이 나타나는 경우

**더 관대하게 (높은 MADs)**
- 희귀 세포 집단을 연구하는 경우
- 데이터셋의 기술적 품질이 높은 경우
- 세포 유형이 자연적으로 극단적인 값을 가지는 경우 (예: 높은 MT%의 뉴런)

**조직별 조정**
- 뇌/뉴런: 더 높은 MT% 임계값이 필요할 수 있음 (10-15%)
- 혈액: MT%를 더 엄격하게 설정 가능 (5-8%)
- 종양 샘플: 생물학적 변이로 인해 종종 더 관대한 임계값이 필요

## 고급 QC 고려사항

### 이 워크플로우에 포함되지 않는 항목

**주변 RNA 보정**
- 도구: SoupX, CellBender, DecontX
- 시기: 드롭렛 기반 데이터에서 높은 배경 RNA가 있는 경우
- 효과: 용해된 세포로부터의 오염 제거

**이중체(Doublet) 탐지**
- 도구: scDblFinder, scrublet, DoubletFinder
- 시기: 드롭렛 기반 데이터에서 항상 권장
- 효과: 다중체(하나의 드롭렛에 2개 이상의 세포) 식별 및 제거

**세포 주기 스코어링**
- 도구: scanpy의 score_genes_cell_cycle
- 시기: 세포 주기 효과가 생물학적 신호를 교란하는 경우
- 효과: 세포 주기 단계를 회귀 제거하거나 고려할 수 있게 함

**배치 보정**
- 도구: Harmony, scVI, ComBat
- 시기: 여러 배치/실험의 데이터를 통합하는 경우
- 효과: 생물학적 특성을 보존하면서 기술적 배치 효과를 제거
