# 자산 만들기

잠재 고객, 대상, 목표에 맞춘 맞춤형 영업 자산을 생성합니다. 인터랙티브 랜딩 페이지, 프레젠테이션 덱, 경영진 원페이저, 워크플로우/아키텍처 데모를 지원합니다.

---

## 트리거

다음의 경우 이 스킬을 호출합니다:
- 사용자가 `/create-an-asset` 또는 `/create-an-asset [회사명]`을 실행
- 사용자가 "자산 만들기", "데모 만들기", "랜딩 페이지 만들기", "워크플로우 목업"을 요청
- 사용자가 영업 대화를 위한 고객 대면 결과물이 필요

---

## 개요

이 스킬은 다음에 대한 맥락을 수집하여 전문적인 영업 자산을 생성합니다:
- **(a) 잠재 고객** — 회사, 연락처, 대화, 페인 포인트
- **(b) 대상** — 누가 보는지, 무엇에 관심이 있는지
- **(c) 목적** — 자산의 목표, 원하는 다음 행동
- **(d) 형식** — 랜딩 페이지, 덱, 원페이저, 또는 워크플로우 데모

그런 다음 조사하고, 구조화하고, 고객에게 공유할 준비가 된 세련된 브랜드 자산을 구축합니다.

## 0단계: 맥락 감지 및 입력 수집

### 0.1단계: 판매자 맥락 감지

사용자의 이메일 도메인에서 어떤 회사에서 일하는지 식별합니다.

| 시나리오 | 조치 |
|----------|--------|
| **단일 제품 회사** | 판매자 맥락 자동 채우기 |
| **다중 제품 회사** | 질문: "이 자산은 어떤 제품이나 솔루션을 위한 것인가요?" |
| **컨설턴트/에이전시/일반 도메인** | 질문: "어떤 회사 또는 제품을 대표하시나요?" |
| **미확인/스타트업** | 질문: "간략히, 무엇을 판매하시나요?" |

### 0.2단계: 잠재 고객 맥락 수집 (a)

**사용자에게 질문:**

| 필드 | 프롬프트 | 필수 |
|-------|--------|----------|
| **회사** | "이 자산은 어떤 회사를 위한 것인가요?" | ✓ 예 |
| **핵심 연락처** | "핵심 연락처는 누구인가요? (이름, 역할)" | 아니오 |
| **거래 단계** | "이 거래는 어떤 단계인가요?" | ✓ 예 |
| **페인 포인트** | "어떤 페인 포인트나 우선순위를 공유했나요?" | 아니오 |
| **과거 자료** | "대화 자료를 업로드해 주세요 (기록, 이메일, 노트, 통화 녹음)" | 아니오 |

**거래 단계 옵션:**
- 소개 / 첫 미팅
- 발견
- 평가 / 기술 검토
- POC / 파일럿
- 협상
- 마감

### 0.3단계: 대상 맥락 수집 (b)

| 필드 | 프롬프트 | 필수 |
|-------|--------|----------|
| **대상 유형** | "누가 보나요?" | ✓ 예 |
| **구체적 역할** | "맞춤화할 구체적 직함이 있나요? (예: CTO, VP Engineering, CFO)" | 아니오 |
| **주요 관심** | "가장 관심 있는 것은?" | ✓ 예 |
| **이의** | "다루어야 할 우려나 이의가 있나요?" | 아니오 |

**대상 유형 옵션:**
- 경영진 (C-suite, VP)
- 기술 (아키텍트, 엔지니어, 개발자)
- 운영 (Ops, IT, 구매)
- 혼합 / 크로스펑셔널

### 0.4단계: 목적 맥락 수집 (c)

| 필드 | 프롬프트 | 필수 |
|-------|--------|----------|
| **목표** | "이 자산의 목표는?" | ✓ 예 |
| **원하는 행동** | "보고 나서 뷰어가 무엇을 해야 하나요?" | ✓ 예 |

### 0.5단계: 형식 선택 (d)

**사용자에게 질문:** "어떤 형식이 가장 좋을까요?"

| 형식 | 설명 | 적합한 용도 |
|--------|-------------|----------|
| **인터랙티브 랜딩 페이지** | 데모, 지표, 계산기가 포함된 다중 탭 페이지 | 경영진 정렬, 소개, 가치 제안 |
| **덱 스타일** | 선형 슬라이드, 프레젠테이션 가능 | 공식 미팅, 대규모 대상 |
| **원페이저** | 단일 스크롤 경영진 요약 | 리브-비하인드, 빠른 요약 |
| **워크플로우 / 아키텍처 데모** | 애니메이션 플로우가 포함된 인터랙티브 다이어그램 | 기술 심층, POC 데모, 연동 |

---

## 1단계: 조사 (적응형)

### 맥락 풍부도 평가

| 수준 | 지표 | 조사 깊이 |
|-------|------------|----------------|
| **풍부** | 기록 업로드, 상세 페인 포인트, 명확한 요구사항 | 가벼움 — 갭만 채우기 |
| **보통** | 일부 맥락, 기록 없음 | 중간 — 회사 + 산업 |
| **빈약** | 회사 이름만 | 심층 — 전체 조사 |

### 항상 조사:

1. **잠재 고객 기본**: 매출, 직원, 핵심 지표, 전략 우선순위
2. **리더십**: CEO, CTO, CIO 이름, 직함, 전략/기술에 대한 최근 인용
3. **브랜드 색상**: 회사 웹사이트에서 추출. 1차 색상, 2차 색상, 액센트 저장.

---

## 2단계: 구조 결정

### 인터랙티브 랜딩 페이지

| 목적 | 추천 섹션 |
|---------|---------------------|
| **소개** | 회사 적합성 → 솔루션 개요 → 핵심 사용 사례 → 왜 우리인지 → 다음 단계 |
| **발견 후속** | 그들의 우선순위 → 어떻게 도와주는지 → 관련 사례 → ROI 프레임워크 → 다음 단계 |
| **기술 심층** | 아키텍처 → 보안 & 규정 준수 → 연동 → 성능 → 지원 |
| **경영진 정렬** | 전략적 적합 → 비즈니스 영향 → ROI 계산기 → 리스크 완화 → 파트너십 |
| **POC 제안** | 범위 → 성공 기준 → 타임라인 → 팀 → 투자 → 다음 단계 |
| **거래 마감** | 가치 요약 → 가격 → 구현 계획 → 조건 → 서명 |

---

### 워크플로우 / 아키텍처 데모

**복잡도별 구조:**

| 복잡도 | 컴포넌트 | 구조 |
|------------|------------|-----------|
| **간단** | 3-5 | 단계 주석이 포함된 단일 뷰 다이어그램 |
| **중간** | 5-10 | 단계별 워크스루가 포함된 확대 가능 캔버스 |
| **복잡** | 10+ | 가이드 투어가 포함된 다중 레이어 뷰 (개요 → 상세) |

**표준 요소:**

1. **제목 바**: `[시나리오 이름] — Powered by [판매자 제품]`
2. **컴포넌트 노드**: 각 시스템에 대한 비주얼 박스/아이콘
3. **플로우 화살표**: 데이터 이동을 보여주는 애니메이션 연결
4. **단계 패널**: 현재 단계를 평이한 언어로 설명하는 사이드바
5. **컨트롤**: 재생 / 일시 정지 / 앞으로 / 뒤로 / 리셋
6. **주석**: 핵심 결정 포인트와 가치 추가에 대한 콜아웃
7. **데이터 미리보기**: 각 단계의 샘플 페이로드 또는 변환

## 3단계: 콘텐츠 생성

### 일반 원칙

모든 콘텐츠는:
- 사용자 입력이나 기록의 **구체적 페인 포인트** 참조
- **잠재 고객의 언어** 사용 — 그들의 용어, 진술된 우선순위
- **판매자 제품** → **잠재 고객 니즈** 매핑을 명시적으로
- 가능한 경우 **증거 포인트** 포함 (사례 연구, 지표, 인용구)
- **맞춤형으로 느껴야, 템플릿이 아닌**

## 4단계: 비주얼 디자인

### 색상 시스템

```css
:root {
    /* === 잠재 고객 브랜드 (1차) === */
    --brand-primary: #[조사에서 추출];
    --brand-secondary: #[추출];

    /* === 다크 테마 베이스 === */
    --bg-primary: #0a0d14;
    --bg-elevated: #0f131c;
    --bg-surface: #161b28;
    --bg-hover: #1e2536;

    /* === 텍스트 === */
    --text-primary: #ffffff;
    --text-secondary: rgba(255, 255, 255, 0.7);
    --text-muted: rgba(255, 255, 255, 0.5);

    /* === 액센트 === */
    --accent: var(--brand-primary);

    /* === 상태 === */
    --success: #10b981;
    --warning: #f59e0b;
    --error: #ef4444;
}
```

## 5단계: 명확화 질문 (필수)

**자산을 구축하기 전에 항상 명확화 질문을 합니다.** 정렬을 보장하고 낭비된 노력을 방지합니다.

### 5.1단계: 이해 요약

먼저 사용자에게 이해한 것을 보여줍니다:

```
"구축 계획은 다음과 같습니다:

**자산**: [형식] [잠재 고객 회사]용
**대상**: [대상 유형] — 구체적으로 [역할 알려진 경우]
**목표**: [목적] → [원하는 행동] 유도
**핵심 테마**: [강조할 2-3가지 주요 포인트]

[워크플로우 데모인 경우, 또한 표시:]
**컴포넌트**: [시스템 목록]
**플로우**: [단계 1] → [단계 2] → [단계 3] → ...
```

### 5.2단계: 표준 질문 (모든 형식)

| 질문 | 이유 |
|----------|-----|
| "비전에 맞나요?" | 이해 확인 |
| "성공하려면 꼭 맞춰야 할 한 가지는?" | 우선순위에 집중 |
| "톤 선호? (대담 & 자신감 / 컨설팅 / 기술적 & 정확)" | 스타일 정렬 |
| "집중적이고 간결, 아니면 포괄적?" | 범위 조정 |

### 5.3단계: 형식별 질문

**최대 2회 질문.** 여전히 모호하면 합리적 선택을 하고 참고: "X로 진행했습니다 — Y를 선호하시면 쉽게 조정 가능합니다."

## 6단계: 구축 및 전달

### 자산 구축

위의 모든 사양을 따릅니다:
1. 2단계 기반 구조 생성
2. 3단계 기반 콘텐츠 생성
3. 4단계 기반 비주얼 디자인 적용
4. 모든 인터랙티브 요소 작동 확인
5. 반응성 테스트 (해당하는 경우)

### 출력 형식

**모든 형식**: 자체 완결 HTML 파일
- 모든 CSS가 인라인 또는 `<style>` 태그
- 모든 JS가 인라인 또는 `<script>` 태그
- 외부 종속성 없음 (Google Fonts 제외)
- 쉬운 공유를 위한 단일 파일

**파일 명명**: `[잠재고객명]-[형식]-[날짜].html`

### 전달 메시지

```markdown
## ✓ 자산 생성: [잠재 고객명]

[자산 보기](computer:///path/to/file.html)

---

**요약**
- **형식**: [인터랙티브 페이지 / 덱 / 원페이저 / 워크플로우 데모]
- **대상**: [유형 및 역할]
- **목적**: [목표] → [원하는 행동]
- **섹션/단계**: [개수 및 목록]

---

**배포 옵션**

고객과 공유하려면:
- **정적 호스팅**: Netlify, Vercel, GitHub Pages, AWS S3 또는 모든 정적 호스트에 업로드
- **비밀번호 보호**: 대부분의 호스트가 제공 (예: Netlify 사이트 보호)
- **직접 공유**: HTML 파일을 직접 전송 — 완전히 자체 완결
- **임베드**: 필요하면 다른 페이지에 iframe으로 포함 가능

---

**맞춤화**

다음을 원하시면 알려주세요:
- 색상이나 스타일 조정
- 섹션 추가, 제거, 재정렬
- 메시징이나 카피 다듬기
- 플로우나 아키텍처 변경 (워크플로우 데모)
- 더 많은 인터랙티브 요소 추가
- PDF나 정적 이미지로 내보내기
```

---

## 7단계: 반복 지원

전달 후 반복 준비:

| 사용자 요청 | 조치 |
|--------------|--------|
| "색상 변경" | 새 팔레트로 재생성, 콘텐츠 유지 |
| "X에 대한 섹션 추가" | 새 섹션 삽입, 플로우 유지 |
| "더 짧게" | 압축, 핵심 포인트 우선 |
| "플로우가 잘못됐어요" | 수정을 기반으로 아키텍처 재구축 |
| "우리 브랜드로 대신" | 잠재 고객 브랜드에서 판매자 브랜드로 전환 |
| "3단계에 더 상세하게" | 해당 섹션만 확장 |
| "PDF로 받을 수 있나요?" | 인쇄 최적화 버전 제공 |

**기억**: 기본값은 잠재 고객 브랜드 색상이지만, 초기 구축 후 자체 브랜드나 중립 팔레트로 조정 가능.

## 품질 체크리스트

전달 전 확인:

### 콘텐츠
- [ ] 잠재 고객 회사명이 전체적으로 올바르게 표기
- [ ] 리더십 이름이 현재 것 (오래되지 않은)
- [ ] 페인 포인트가 입력/기록을 정확히 반영
- [ ] 판매자 제품이 정확히 표현됨
- [ ] 플레이스홀더 텍스트가 남아있지 않음
- [ ] 증거 포인트가 정확하고 출처 있음

### 비주얼
- [ ] 브랜드 색상이 올바르게 적용됨
- [ ] 모든 텍스트가 읽기 쉬움 (대비)
- [ ] 애니메이션이 부드럽고 산만하지 않음
- [ ] 모바일 반응형 (인터랙티브 페이지인 경우)
- [ ] 다크 테마가 세련되게 보임

### 기능
- [ ] 모든 탭/섹션이 올바르게 로드됨
