# /forecast

> 익숙하지 않은 플레이스홀더가 보이거나 연결된 도구를 확인해야 하는 경우, [CONNECTORS.md](../../CONNECTORS.md)를 참조하세요.

리스크 분석과 커밋 추천이 포함된 가중 영업 예측을 생성합니다.

## 사용법

```
/forecast [기간]
```

다음에 대한 예측을 생성합니다: $ARGUMENTS

파일이 참조된 경우: @$1

---

## 작동 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                        예측                                      │
├─────────────────────────────────────────────────────────────────┤
│  단독 사용 (항상 작동)                                            │
│  ✓ CRM에서 CSV 내보내기 업로드                                   │
│  ✓ 또는 파이프라인 거래를 붙여넣기/설명                           │
│  ✓ 할당 및 타임라인 설정                                          │
│  ✓ 단계 확률이 포함된 가중 예측 제공                              │
│  ✓ 리스크 조정 예측 (최선/예상/최악)                              │
│  ✓ 커밋 vs. 업사이드 분해                                        │
│  ✓ 갭 분석 및 추천                                               │
├─────────────────────────────────────────────────────────────────┤
│  강화 (도구를 연결하면)                                            │
│  + CRM: 파이프라인 자동 가져오기, 실시간 데이터                   │
│  + 단계, 세그먼트, 거래 규모별 과거 승률                          │
│  + 리스크 점수를 위한 활동 신호                                    │
│  + 자동 갱신 및 시간에 따른 추적                                   │
└─────────────────────────────────────────────────────────────────┘
```

## 필요한 정보

### 1단계: 파이프라인 데이터

**옵션 A: CSV 업로드**
CRM (예: Salesforce, HubSpot)에서 파이프라인을 내보냅니다. 최소 필요:
- 거래/기회 이름
- 금액
- 단계
- 마감일

도움이 되는 항목:
- 담당자 (팀 예측인 경우)
- 마지막 활동 날짜
- 생성 날짜
- 계정 이름

**옵션 B: 거래 붙여넣기**
```
Acme Corp - $50K - 협상 - 1월 31일 마감
TechStart - $25K - 데모 예정 - 2월 15일 마감
BigCo - $100K - 발견 - 3월 30일 마감
```

**옵션 C: 영역 설명**
"8개 거래가 파이프라인에 있고 총 $400K입니다. 2개는 협상 중 ($120K), 3개는 평가 중 ($180K), 3개는 발견 중 ($100K)."

### 2단계: 목표

- **할당**: 목표는 얼마인가? (예: "이번 분기 $500K")
- **타임라인**: 기간이 언제 끝나는가? (예: "Q1은 3월 31일 종료")
- **이미 마감**: 이 기간에 이미 예약한 금액은?

## 단계 확률 (기본값)

사용자 지정 확률을 제공하지 않으면 다음을 사용합니다:

| 단계 | 기본 확률 |
|-------|---------------------|
| 마감 성공 | 100% |
| 협상 / 계약 | 80% |
| 제안 / 견적 | 60% |
| 평가 / 데모 | 40% |
| 발견 / 자격 검증 | 20% |
| 잠재 / 리드 | 10% |

단계나 확률이 다르면 알려주세요.

## CRM이 연결된 경우

- 파이프라인을 자동으로 가져옵니다
- 실제 과거 승률을 사용합니다
- 리스크 점수에 활동 최근성을 반영합니다
- 시간에 따른 예측 변화를 추적합니다
- 이전 예측과 비교합니다

---

## 팁

1. **커밋에 솔직하세요** — 확실히 걸 수 있는 거래만 커밋. 업사이드는 나머지 모두.
2. **마감일을 업데이트하세요** — 오래된 마감일은 예측 정확성을 죽입니다. 제시간에 마감되지 않을 거래는 연기.
3. **커버리지가 중요** — 3배 파이프라인 커버리지가 건강. 2배 미만은 위험.
4. **활동 = 신호** — 최근 활동이 없는 거래는 단계가 시사하는 것보다 더 높은 리스크.
