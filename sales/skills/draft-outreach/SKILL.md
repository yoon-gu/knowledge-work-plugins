# 아웃리치 초안

먼저 조사하고, 그 다음 초안합니다. 이 스킬은 제네릭 아웃리치를 절대 보내지 않습니다 — 항상 먼저 잠재 고객을 조사하여 메시지를 개인화합니다. 웹 검색으로 단독 작동하며, 도구를 연결하면 강화됩니다.

## 커넥터 (선택)

| 커넥터 | 추가되는 것 |
|-----------|--------------|
| **강화** | 확인된 이메일, 전화, 배경 세부 |
| **CRM** | 이전 관계 맥락, 기존 연락처 |
| **이메일** | 받은편지함에 직접 초안 생성 |

> **커넥터 없나요?** 웹 조사가 훌륭하게 작동합니다. 이메일 텍스트를 출력해 복사하실 수 있습니다.

---

## 작동 방식

```
+------------------------------------------------------------------+
|                      아웃리치 초안                                  |
|                                                                   |
|  1단계: 조사 (항상 먼저)                                           |
|  - 웹 검색 (기본)                                                  |
|  - + 강화 (강화 도구 연결 시)                                      |
|  - + CRM (CRM 연결 시)                                            |
|                                                                   |
|  2단계: 초안 (조사 기반)                                           |
|  - 개인화된 오프닝 (조사에서)                                      |
|  - 관련 후크 (그들의 우선순위)                                     |
|  - 명확한 CTA                                                     |
|                                                                   |
|  3단계: 전달 (커넥터 기반)                                         |
|  - 이메일 초안 (이메일 연결 시)                                    |
|  - LinkedIn용 카피 (항상)                                          |
|  - 사용자에게 출력 (항상)                                          |
+------------------------------------------------------------------+
```

### 3단계: 후크 식별


## 이메일 스타일 가이드라인

1. **간결하지만 정보적** — 빠르게 핵심으로. 바쁜 사람은 훑어 읽습니다.
2. **마크다운 서식 없음** — 별표, 굵게 (**텍스트**) 등 마크다운을 절대 사용하지 마세요. 모든 이메일 클라이언트에서 자연스러운 일반 텍스트로.
3. **짧은 문단** — 문단당 최대 2-3문장. 공백이 친구입니다.
4. **간단한 목록** — 항목을 나열할 때 일반 대시 사용. 화려한 서식 없음.

## 하지 말아야 할 것

**제네릭 오프닝:**
- "이 이메일이 잘 전달되길 바랍니다"
- "연락드리는 이유는..."
- "자기 소개를 하고 싶었습니다"

**기능 나열:**
- 제품에 대한 긴 문단
- 한 번에 여러 가치 제안
- 명확한 CTA 없음

**가짜 개인화:**
- "[회사]에서 일하시는 것을 봤습니다" (당연히)
- "역할을 축하드립니다" (맥락 없이)

**이메일에 마크다운:**
- **굵게**나 *기울임* 별표 사용
- 렌더링되지 않는 헤더나 서식 목록

**대신:**
- 배운 구체적인 것으로 시작
- 하나의 명확한 가치 제안
- 하나의 명확한 요청
- 일반 텍스트 서식만

## 채널 선택

```
확인된 이메일이 있으면:
  → 이메일 선호 (더 높은 응답률)
  → LinkedIn 백업도 제공

이메일이 없으면:
  → LinkedIn 연결 요청
  → 연결 후 후속 메시지 템플릿

웜 소개가 가능하면:
  → 먼저 상호 연결을 통한 아웃리치 제안
```
